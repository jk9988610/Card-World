# Internationalization (i18n)

## Phase 0 (now)

- UI strings: `locales/en.json`, `locales/zh-Hans.json`
- Switch: `?lang=zh-Hans` or `?lang=en` (header links)
- Loader: `js/i18n.js` with `data-i18n` on HTML

## Later (card content)

In definitions, use keys instead of inline English:

```json
{
  "slug": "content.door",
  "titleKey": "cards.door.title",
  "textKey": "cards.door.text"
}
```

Add `cards.door.title` / `cards.door.text` to each locale file. Resolve at render time in the card renderer.

## Adding a language

1. Copy `locales/en.json` → `locales/<code>.json`
2. Add code to `supported` in `js/i18n.js`
3. Add a link in `index.html`
